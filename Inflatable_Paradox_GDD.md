# 《充气悖论》(Inflatable Paradox) 完整版游戏设计文档(GDD)

本文档为全量闭环、可直接落地开发的工业级GDD，已完成全流程规则校验与逻辑漏洞修复。完全对齐Game Jam开发周期、Unity作品集技术展示目标与梦核风格定位。

## 1. 核心概述 (Overview)

### 1.1 基础信息
| 项目 | 详情 |
|---|---|
| 游戏名称 | 充气悖论 (Inflatable Paradox) |
| 游戏类型 | 本地双人同屏 3D 动作竞技派对游戏 (Party Brawler) |
| 目标平台 | Windows PC |
| 开发引擎 | Unity 2022.3+ URP管线 |
| 单局时长 | 60秒极速一命制对局 |
| 核心视觉风格 | 梦核 (Dreamcore) / 泳池核 (Poolcore)，高明度低饱和度马卡龙色系（主色：柔和粉、泳池蓝、奶白、浅青） |
| 开发核心目标 | 作为Unity客户端开发作品集，核心展示底层架构（FSM+Event Bus）、渲染性能（MaterialPropertyBlock/顶点形变），不追求冗余技能数量 |

### 1.2 核心设计理念
锚定「充气悖论」核心主题：占领地块越多，玩家角色体积越大，受击判定框越广。 玩家必须在“扩张领地拿资源”与“消耗地块缩小体积求生存”之间高频决策。所有玩法机制围绕该主题设计，拒绝视觉与玩法脱节，打造「低门槛上手，高上限博弈」的派对竞技体验。

### 1.3 赛制规则
* **核心规则**： 一命淘汰制（1-Stock Elimination），玩家被击飞出斗技台边界即判定淘汰，对方获胜。
* **时长规则**： 单局固定60秒，倒计时结束双方均未淘汰时，按己方领地地块占比判定胜负；占比完全一致时，直接判定为平局。
* **对局规则**： 支持本地双人无限续局，无局外成长系统，单局内所有资源、状态对局结束后清零。

## 2. 核心玩法与循环设计

### 2.1 三层核心循环设计
| 循环层级 | 核心内容 | 设计目标 |
|---|---|---|
| 乐趣原子单位（玩家每秒操作） | 移动瞄准→主动按键喷射颜料涂色/反涂地块→即时获得地块控制权与充气资源反馈 | 让玩家每一秒都有明确操作目标，即时反馈拉满，无空窗期 |
| 单局核心循环 | 占领地块（角色体积变大/风险升高）→ 消耗地块释放冲击波（击飞对手/自身瘦身降风险）→ 地形压迫+累计受伤值放大击飞效果→ 击杀淘汰 | 形成闭环博弈，每一步操作都有明确收益与对应风险，完全贴合「充气悖论」物理主题 |
| 元游戏（玩家每次会话目标） | 本地双人连胜挑战、刷新最短击杀时长本地记录、尝试不同博弈套路 | 解决派对游戏复玩率问题，无需复杂局外系统即可提供长线目标 |

### 2.2 MDA框架机制-动态-美学对应表
| 核心机制 | 玩家行为动态 | 目标美学体验 |
|---|---|---|
| 充气涂色+物理体积膨胀 | 玩家在「扩张领地拿资源」和「消耗地块瘦身降风险」之间反复博弈，不敢无脑涂满全图 | 悖论式决策乐趣、持续的风险对冲紧张感 |
| 分段式领地冲击波（消耗地块） | 玩家需要权衡「用多少资源换攻击强度」，满蓄力高风险操作带来高回报，同时也起到“放气”的作用 | 即兴发挥的爽感、赌徒式博弈乐趣 |
| 分级闪避+完美闪避攻防逆转 | 新手靠普通闪避保命，硬核玩家靠完美闪避实现绝地翻盘，全层级玩家都有操作空间 | 操作成就感、逆风翻盘的情绪峰值 |
| 分级动态地形塌陷 | 玩家操作空间持续压缩，从试探到决战的节奏清晰，杜绝无效平局拉扯 | 持续升级的压迫感、极速决战的肾上腺素飙升 |
| 累计受伤值运动学击飞系统 | 鼓励玩家持续进攻，受伤值越高抛物线越远，避免「满血刮痧」 | 进攻正反馈、压制与翻盘的双向博弈空间 |
| 充气软体质感顶点Shader | 玩家移动、涂色时地块有实时形变反馈，强化「充气」主题，而非冰冷的方块 | 梦核风格的诡异柔软感、沉浸式主题体验 |

### 2.3 巴特尔玩家分类适配
* **社交者（核心服务对象）**： 低操作门槛、短单局时长、强情绪波动。
* **杀手**： 冲击波击飞、地形杀、连续压制。
* **成就者**： 最短击杀时长、本地排行榜。
* **探索者**： 差异化的地图掩体布局、塌陷节奏变体。

## 3. 关卡与环境设计

### 3.1 物理交互区（核心斗技台）
* **尺寸与形状**： 紧凑的 11x11 切角网格阵列，有效可站立网格121个。开局提供合理的战略缓冲空间，提升前期容错率。
* **核心空间结构**：
  * **中央微高台**： 比基础平面高0.5单位，提供物理击飞抛物线优势，无遮挡。
  * **不可破坏充气立柱**：共4根，分布于斗技台四角。绝对静止的刚体。玩家无法使其移动，需利用其物理碰撞体来阻断敌方冲击波的判定视线。
  * **地块网格**： 统一1x1正方形，通过MapGenerator批量生成，支持O(1)效率查询。

### 3.2 视觉远景区（深渊乐园）
* **核心设计**： 斗技台下方和远景深渊中放置巨大的充气滑梯、摩天轮等装饰模型（玩家绝对无法交互）。
* **氛围营造**： 迷雾Shader软化远景，弱化细节、降低饱和度。
* **性能约束**： 远景总面数≤5000三角面，带2级LOD，无碰撞体，关闭实时光照。

### 3.3 60秒动态地形塌陷节奏
| 时间段 | 阶段名称 | 地形变化规则 | 核心玩法引导 |
|---|---|---|---|
| 0-20秒 | 充气试探期 | 完整11x11地图，无塌陷，所有地块可站立 | 玩家自由抢地，积累充气资源，熟悉对手操作节奏 |
| 20-35秒 | 空间施压期 | 最外围一圈网格进入「漏气预警」（红光闪烁+漏气音效），35秒整一次性坠入深渊 | 玩家权衡「继续抢外围」还是「退守内圈」，提前规划安全区 |
| 35-50秒 | 充气决战期 | 第二圈网格进入漏气预警，50秒整一次性坠入深渊，地图区域大幅收缩 | 玩家必须正面博弈，用冲击波压缩对方生存空间，放大受伤值优势 |
| 50-60秒 | 极速猝死期 | 每隔1.5秒，随机掉落1块非己方占领的地块，直到一方淘汰 | 鼓励玩家占领更多地块获得安全区，杜绝平局拉扯 |

## 4. 核心玩法机制详解

### 4.1 核心状态与动态体积博弈（核心悖论）
* **地块状态规则**： Neutral（白色中立）、Player1（青蓝色）、Player2（肉粉色）。
* **反占领成本**： 中立地块单次命中0.15秒涂满，敌方地块0.25秒涂满。
* **体积膨胀法则**： 角色基础体积（Scale）为1.0。场上己方地块比例每增加10%，角色体积和碰撞体放大5%，最高放大至1.5倍。体积越大，被敌方颜料和冲击波命中的概率呈几何级上升。

### 4.2 主动喷射涂色机制
* **无限喷射（无弹药槽）**：颜料喷射没有任何资源或冷却限制。玩家的注意力应全部集中在走位和地块颜色的抢夺上，涂色本身就是获取冲击波弹药的手段。
* **精度与衰减**：最大有效射程3格。奔跑状态喷射射程衰减1格，散射扩大50%；行走/站立状态为精准线性弹道。

### 4.3 攻防博弈系统

#### 4.3.1 分段式领地冲击波
* **操作逻辑**： 按住重击键蓄力，吸收脚下及相邻己方地块（最多8格），松开释放冲击波。蓄力过程中锁定移动、喷射、闪避。释放后角色体积根据消耗地块立即缩小。
* **强度公式（硬性阈值）**：
  * **Lv.1（吸收1-3格）**： 基础排斥力，击退距离较短。
  * **Lv.2（吸收4-6格）**： 中等冲击力，附带0.5秒微弱硬直。
  * **Lv.3（吸收7-8格）**： 极限爆发，极强击飞排斥力，判定半径最大化。
* **相杀机制**： 双方极短时间内（±0.1秒）释放，触发中立爆炸，按冲击波段位比例等距推开。

#### 4.3.2 分级闪避系统
* **普通闪避**： 地面短按，0.2秒无敌帧，固定位移，冷却0.8秒。
* **完美闪避**： 冲击波命中前0.15秒内按下。无视冲击，触发0.5秒子弹时间，瞬间将脚下九宫格同化为己方颜色。

#### 4.3.3 累计受伤值系统（抛物线基数）
* **核心规则**： 初始受伤值0%，每次被冲击波命中累加（Lv.1+15%，Lv.3+120%）。
* **击飞公式**： 击飞距离 = 冲击波基础推力 × (1 + 受伤值/100)^1.5
* **衰减规则**： 命中后锁定3秒，随后每秒衰减2%。淘汰清零。

## 7. 音效与视听错位设计规范 (Audio Detachment)
核心原则：摒弃传统动作游戏的热血反馈，用诡异、冷漠的声音包装高烈度对战，放大心理压迫感。

| 音效分类 | 触发逻辑 | 风格参数要求 | 音量基准 |
|---|---|---|---|
| 颜料喷射 | 持续播放 | 沉闷的塑胶摩擦声 | 0.3 |
| 地块涂满 | 单次播放 | 类似踩扁空塑料瓶的闷响 | 0.4 |
| 冲击波蓄力 | 循环播放 | 极低频的空气压缩声（带耳鸣感） | 0.3-0.6 |
| 冲击波释放 | 单次播放 | 巨大的、沉重的“叹气声”或轮胎爆胎声 | 0.4-0.7 |
| 完美闪避 | 单次播放 | 空旷泳池里的一滴水滴声（带极强混响） | 0.6 |
| 受伤击飞 | 单次播放 | 走调的八音盒发条声或干瘪的橡胶尖叫 | 0.5 |
| 塌陷预警 | 循环播放 | 指甲刮擦气球的刺耳声 | 0.3 |
| 全局氛围音 | 循环播放 | 剥离常规BGM，仅保留失真的商场广播噪音或死寂的白噪音 | 0.15 |

* **淘汰表现**：玩家掉入深渊时，不触发任何华丽的爆炸或礼花特效，仅留下一道干瘪的坠落音效，迅速被远景迷雾吞噬。

## 8. 技术架构与实现规范

### 8.1 核心架构原则
严禁使用MVVM/MVP架构。全程采用事件驱动（Event Bus）+ 全局管理器（Manager）模式解耦。
* 初始化顺序： LevelManager → Event Bus → InputManager → TerritoryManager → PlayerManager → SoundManager → UIManager。

### 8.2 玩家状态与运动学击飞 (核心修正点)
* **FSM状态机**： 独立封装 Idle, Move, Spray, Charge, Dodge, Hurt, Knockback, Eliminated。同一时间仅允许1个核心状态。
* **抛物线运动学方案（替代Ragdoll）**：
  * **彻底废弃原有的物理布娃娃系统**，避免在碰撞复杂的边界发生不受控的“原力弹射”。
  * 角色受击直接进入 Hurt/Knockback 状态，播放基于Animator的固定受击翻滚动画片段。
  * 系统利用初速度、发射角度、重力常数以及玩家当前的受伤值系数，预结算出飞行轨迹，随后在Update中强制更新Transform，确保每一次击飞的落点完全符合数值预期。

### 8.3 数据管理
核心配置（地块耗时、冲击波各段位强度阈值、角色Scale放大倍率参数）全部抽离至 Config_XXX (ScriptableObject)。运行态数据由管理器独占修改权并暴露只读接口。本地数据（最短击杀）通过PlayerPrefs存储。
