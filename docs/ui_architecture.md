# 童核风格 UI 架构与设计规范

更新时间：2026-05-04

## 1. UI 出入栈结构 (UI Stack Architecture)

建议使用经典的**栈式管理 (Stack-based UI Manager)**，每次打开新界面就 `Push()`，关闭或返回就 `Pop()`。

```text
UIManager (单例/系统)
 ├── [常驻底层] GameHUDPanel      (对战主界面：受伤值、倒计时)//领地占比进度条疑似并非
 ├── [全屏] MainMenuPanel     (开始游戏、设置、退出)
 ├── [全屏] ResultPanel       (游戏结算、胜负展示、重来/返回主界面)
 ├── [弹窗] PauseMenuPanel    (游戏中按 Esc 呼出，覆盖在 HUD 之上)
 └── [弹窗] SettingsPanel     (设置界面，可以从主界面或暂停界面呼出)
```

**出入栈逻辑示例：**
- 游戏主菜单：栈底是 `MainMenuPanel`。
- 开始游戏：`Pop` 掉主菜单，`Push` 进 `GameHUDPanel`。
- 按下暂停 (Esc)：不销毁 HUD，直接 `Push` 进 `PauseMenuPanel`。
- 此时按设置：`Push` 进 `SettingsPanel`（设置在最上层）。
- 退出设置：`Pop` 掉设置，露出 `PauseMenuPanel`。

## 2. 童核 (Kid-core) 美术风格规范

童核风格旨在唤起童年怀旧、游乐场、塑料玩具和贴纸的感觉。

### 色彩搭配
- **高饱和度基础色**：大红、明黄、天蓝、草绿（参考乐高、波波池、充气城堡）。//贴纸真的有必要这样吗。。
- **材质感**：避免写实的金属或粗糙表面，UI 应该看起来像**光滑的塑料**、**软胶**或**贴纸**。
- **背景/对比**：使用高对比度的颜色，不要用暗黑色的半透明遮罩，暂停界面的背景模糊可以加一点彩色色调。

### 形状与轮廓
- **圆润饱满**：按钮和面板使用大圆角（完全没有尖锐的直角）。
- **贴纸描边 (Sticker Outline)**：给 UI 元素（特别是图标和文字）加上粗大的白色描边，让它们看起来像剪下来的贴纸。
- **立体感**：使用明显的向下偏移投影（Drop Shadow），但不做羽化模糊，做成硬边缘的投影（例如向下平移 5px 的纯色色块），营造出厚纸板或塑料块的立体感。

### 字体 (Typography)
- 放弃细长的严肃字体，选择**粗体、圆润、带有卡通感**的无衬线字体。//或者涂鸦手写风
- 推荐开源字体参考：`Fredoka One`, `Baloo`, `Varela Round`

### 动效 (Juiciness)
- **Q弹反馈**：按钮按下时不要只是变色，应该缩放变小（Scale 1.0 -> 0.9），松开时反弹（0.9 -> 1.1 -> 1.0）。
- **入场动画**：面板弹出时像果冻一样从中间放大并抖动。

## 3. 各界面内容清单

### MainMenuPanel (主界面)
- 大号 LOGO（带果冻动画）
- "开始游戏" 按钮（最大最醒目）
- "设置" 按钮
- "退出" 按钮

### GameHUDPanel (对战界面)
- **顶部居中**：倒计时时钟
- **顶部两边或底部**：双方的涂色进度条//有必要吗？
- **底部左右**：P1/P2 的受伤量/buff/debuff状态图标

### PauseMenuPanel (暂停界面)
- "继续游戏" 按钮
- "设置" 按钮
- "放弃并返回主界面" 按钮

### SettingsPanel (设置界面)
分为三个页签（或者从上到下排列）：
1. **画面 (Graphics)**
   - 全屏/窗口模式开关。
   - **CRT 滤镜强度**：滑动条（0~100%），可以随时预览怀旧电视效果。
2. **音频 (Audio)**
   - 主音量滑动条
   - 音乐滑动条
   - 音效滑动条（调整时播放一声玩具鸭子的"嘎"声作为反馈）
3. **控制 (Controls)**
   - P1 / P2 键位映射查看与修改功能。

### ResultPanel (结算界面)
- "胜利/失败" 的夸张大字。
- 喷射总面积统计数字的滚动动画。
- "再来一局" 按钮。
- "返回主菜单" 按钮。

## 4. 后续接入指南
- **UI 预制体**：建议把每个 Panel 做成独立的 Prefab。
- **Input System 桥接**：在 UI 状态时，通过 `EventSystem` 和 `InputModule` 接管输入，暂停后台玩家的移动读取。
