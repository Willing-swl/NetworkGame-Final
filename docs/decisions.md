# 决策记录

## ADR-0001: 以 GDD copy 作为当前基线
- 状态：已接受
- 决策：以 [Inflatable_Paradox_GDD copy.md](../Inflatable_Paradox_GDD%20copy.md) 作为当前 canonical baseline。
- 理由：这版已经明确了当前的核心方向，后续文档和实现都应该围绕它展开，而不是让历史版本继续干扰当前开发。
- 后果：后续的风格变化、主题变化或数值重构，应该以 delta 形式记录。

## ADR-0002: 运行时架构保持 Event Bus + Manager + FSM
- 状态：已接受
- 决策：当前项目的核心技术骨架保持事件总线、系统管理器和玩家 FSM 的组合。
- 理由：代码里已经存在稳定的 `PrototypeMatchManager`、`PrototypeInputManager`、`PrototypePlayerManager`、`PrototypeGridManager`、`PlayerController` 和 `EventManager`，继续沿这条路扩展成本最低。
- 后果：不引入 MVVM 一类会反向耦合玩法逻辑的结构。

## ADR-0003: 主题变更走 delta，不回写基线
- 状态：已接受
- 决策：梦核到童核之类的美术或氛围变化，统一归档到 variants 或 delta 文档。
- 理由：基线文档需要保持稳定，才能让后来的 AI 和人快速判断“当前项目原始规范是什么”。
- 后果：改风格时先写变更说明，再同步影响到相关的架构、音频或后处理文档。

## ADR-0004: 会话记忆只做临时工作区
- 状态：已接受
- 决策：会话记忆只保存当前任务的中间计划和临时上下文，不作为长期知识库。
- 理由：真正能跨会话传递的，应该是仓库里的文档和 repo memory。
- 后果：每次结束工作时，都要把结论落到 docs 或 repo memory。

## ADR-0005: 冲击波脱离硬性阈值改为平滑公式
- 状态：已接受
- 决策：弃用原 GDD 的“1~3格1级，4~6格2级”硬件分段设定。改用连续判定：`吸收格数 / 最大配置格数`。
- 理由：连通图机制的乐趣在于“每多一格都赚”，硬分段会弱化玩家努力涂边角格的激励；同时空放(吸到0格)会被旧判定强转成1级，导致白嫖。
- 后果：参数计算已更新为 `Mathf.Lerp` 平滑缩放，`ReleaseShockwave` 新增拦截（<=0时禁止释放冲击波）。

## ADR-0006: 喷涂占领从渐变进度改为瞬间占领
- 状态：已接受
- 决策：从 `TryApplySpray` 中移除了需要 0.15s/0.25s 的进度渐变延迟（通过将延时参数压高到10f），实现“一喷即100%占领”。
- 理由：原来的逻辑与视觉强烈脱节——Shader 读到 1% 的 progress 就会覆盖蓝色，而底层的吸收逻辑死磕 100%，导致玩家以为已经占领了蓝色地块却无法用来发动冲击波（变成中立反白 Bug）。
- 后果：游戏节奏变快，去除了软泥拉锯感，防守方的反应窗口缩短。后续需要从其它维度进行平衡。
